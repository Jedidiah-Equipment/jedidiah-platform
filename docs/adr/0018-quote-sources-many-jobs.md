# A Quote sources any number of Jobs

> **Status: to be revisited.** This ADR predates the Quote-status rework in ADR 0006. The eligibility text below gates Job sourcing on Quote status (`draft|sent|accepted` allowed, `rejected` excluded), but status is now a cosmetic label with no side effects. The Job-side gating model needs to be redesigned when Jobs come back in scope; until then, treat status-based eligibility as superseded but the one-Quote-to-many-Jobs cardinality decision as still standing.

The platform originally enforced one Job per Quote (a `job_quote_id_unique` index, and CONTEXT.md "Quote Conversion"). A single sales Quote can in practice lead to several physical builds, so we have dropped that constraint: a Quote in `draft`, `sent`, or `accepted` status may be the source of any number of Jobs (zero or more); only `rejected` Quotes are excluded. The `sent` status was also added to the eligible set, and the requirement that a Quote carry a Product before it can source a Job was removed — the Job's Product is chosen in the Create-Job dialog instead.

This reverses a deliberate 1:1 decision. The 1:1 model gave a clean "this Quote = this Job" mental model and a cheap database guarantee; the trade-off is that Quote→Job is now a true one-to-many relationship, which the Quotes list and `QuoteSummary` reflect by carrying a list of Jobs rather than a single `jobCode`/`jobId`. Re-introducing the 1:1 constraint later would require data cleanup once multiple Jobs exist per Quote. The term "Quote Conversion" is retired in favour of "Create Job from Quote" — a Quote is never consumed or transformed, it is the optional source of Jobs.
