# kibo-ui is the vendored foundation for the Schedule Gantt

The Schedule Gantt needs a timeline grid with drag-and-resize bar mechanics — the genuinely hard part to build well. Rather than build that bespoke, we adopt kibo-ui's Gantt component via `npx kibo-ui add gantt`, which vendors the component source into the repo (shadcn-registry style, like `pkg/web/src/components/ui/*`). We own the vendored source and adapt it for the domain: a daily grid (kibo-ui defaults to monthly), the plan-vs-actual overlay, the Job-pinned three-level structure, and the cross-job Station Contention overlay.

The trade-off: kibo-ui pulls in **dnd-kit**, **lodash**, and **jotai**. jotai is a second client-state library alongside the project's zustand — a future reader will see it in `pkg/web/package.json` and wonder why. It is accepted because it is confined entirely to the one vendored Gantt component; it is not a project-wide state choice. The rejected alternative — a bespoke Gantt — would fit the existing stack (Base UI, zustand) with no new dependencies, but would reinvent the drag-on-grid mechanics that are kibo-ui's whole value. Swapping the foundation later would mean rebuilding the domain Gantt layered on top of it.
